**The `nan` issue:**

Remember our reward formula:
```
reward = (portfolio_value - prev_portfolio) / prev_portfolio
```

This is division. What if `prev_portfolio` somehow becomes 0?

🍕 Dividing 10 pizzas among 0 people = impossible = `nan`

That's exactly what happened at Episode 50. Portfolio hit zero at some point.

---

**How did I spot it?**

Simple — I saw `nan` in the output and immediately thought *"division by zero somewhere"*. The only division in our code is the reward formula. So that's where the fix goes. 😄

---

**Where exactly to add the fix:**

Go to your **TradingEnv cell** (Step 2 cell). Find these two lines:

```python
# Reward = % change in portfolio value
reward = (self.portfolio_value - prev_portfolio) / prev_portfolio
```

Change them to:

```python
# Reward = % change in portfolio value
reward = (self.portfolio_value - prev_portfolio) / prev_portfolio
reward = np.nan_to_num(reward, nan=0.0)
```

`nan_to_num` simply says — *"if the result is nan, replace it with 0"*. Safe landing! 😄

**Epsilon decay — your understanding is almost right!**

But it's not per episode — it's per **step** (per trading day).

So more like:
- Day 1 → epsilon 1.0 → 100% random
- Day 2 → epsilon 0.995 → 99.5% random
- Day 3 → epsilon 0.990 → 99% random
- ...keeps decaying every single day...
- Day 748 → epsilon very small → mostly smart decisions

That's why by Episode 0 epsilon was already 0.029 — it had already decayed through 748 days! 😄
